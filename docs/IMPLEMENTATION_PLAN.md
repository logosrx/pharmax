- [x] `@pharmax/crypto` â€” envelope encryption with per-tenant KEK, **landed before any PHI table**. Implements the full primitive set the Phase 2 patient/prescription/order tables will depend on. **Canonical AAD encoding** (`crypto.v1` versioned, NUL-separated, sorted-key) over the record binding `{tenantId, table, column, recordId}` â€” every binding field demonstrably influences the AAD so a ciphertext moved between rows (different recordId, different column, different table, or different tenant) fails authentication on decrypt; empty fields and embedded NUL bytes are rejected at encode time. **Ciphertext envelope** is a JSON object (`{v: 1, alg: "AES-256-GCM", kek, wDek, iv, ct, tag}`) stored as a Prisma `Json` column; `parseEnvelope` is the choke point that rejects malformed shapes before the cipher sees them, and `ct === ""` is legitimate (zero-byte plaintext for empty PHI fields like `address_line_2`). **`KmsAdapter`** interface separates the seam between this package and the key store; production binds AWS KMS, dev/test binds `LocalKmsAdapter` which derives per-tenant KEKs via HKDF-SHA-256 (`info = "pharmax.kek.v1.<tenantId>.v<n>"`), rotates KEKs in-place via `rotateKek({tenantId})` (kid version bumped; historical wrapped DEKs continue to unwrap â€” the rotation guarantee), and derives per-tenant per-purpose 32-byte search keys (`info = "pharmax.search.v1.<tenantId>.<purpose>"`) that are decoupled from KEK lifecycle so blind-index columns survive KEK rotation.
